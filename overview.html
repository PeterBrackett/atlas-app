Opportunity scorecard — Overall by segment

Dimensions included in Overall score
How this works Unchecking a dimension excludes it from every segment's Overall score calculation (and from the "Scored X/N" count) everywhere — the overview matrix, every country page, the project builder, and Word/PowerPoint exports. Its column/row stays visible, just doesn't count toward Overall or gate it. This is global and persisted, not a personal filter: everyone who loads Atlas sees the same enabled/disabled set until it's changed again here. Turning a dimension back on may make some segments' Overall go blank again if that dimension isn't actually scored for them.
Bulk auto-scoring tools (market opportunity, regulatory complexity, distribution resources required, languages required, local presence required, client servicing, consultant reliant, pricing impact, outsourced management, comingled vehicles, investor decision-making)
Auto-score "Market opportunity" from an allocation:
How this works Scores every segment in every loaded country at once, based on the asset class picked here (independent of the "Show:" selector below): below threshold = 1, at or above threshold = 2, at or above optimal = 3. As of 2026-07-15 this scores off the extrapolated max figure (reported allocation scaled up for institutions that never filed an asset-class breakdown — see the "range (min–max)" column on each country page), not the raw reported figure, so a segment with a lot of non-reporting institutions will score higher than it used to. A segment with no allocation data at all for this selection is treated as $0 (scores 1, unless threshold is 0 or less) — there's nothing to extrapolate from. Overwrites any existing Market opportunity score. This panel is always here whenever you're signed in — it no longer depends on which view is selected below.
Auto-score "Regulatory complexity" from TMF Group rankings ():
How this works TMF's rank 1 is the most complex jurisdiction to do business in, rank 79 the least — this is inverted from the allocation scorer above since low complexity is the attractive case: rank at or below threshold = 1, above threshold = 2, at or above optimal = 3. Countries TMF doesn't cover (Iceland, as of the 2025 edition) are skipped and left as-is — reported after applying. Same segment-independent country-level value applied to every segment, and overwrites any existing Regulatory complexity score.
Auto-score "Distribution resources required" from institution concentration:
How this works Uses each segment's top-3-institution share of AUM (from the same Company-file institution data segment totals are built from) as a proxy for how much distribution resource covering the market would take: below threshold = 1, at or above threshold = 2, at or above optimal = 3 — a handful of large institutions dominating a segment needs less coverage effort than AUM spread across thousands of small ones. Segments with no institution-level concentration data (e.g. Life/Non-life insurance, built from industry aggregates rather than individual institutions) are skipped, not guessed at. Overwrites any existing Distribution resources required score.
Auto-score "Languages required" from EF English Proficiency Index ():
How this works EF EPI score runs 0-800, higher = more of the population speaks English well — direction matches the allocation scorer above (not inverted): below threshold = 1, at or above threshold = 2, at or above optimal = 3. The US, UK, Ireland, Canada, Australia and New Zealand aren't ranked by EF EPI at all (it only measures non-native proficiency), so they're automatically scored 3 — English is the native language there by definition. Singapore is also unranked by EF EPI but is scored 3 as a judgment call, since English is Singapore's dominant language of government, courts and business; override this per-segment if you'd rather treat it differently. Iceland has no EF EPI coverage and isn't an English-native/business market, so it's genuinely uncovered and skipped, same as Iceland's Regulatory complexity gap. Same country-level value applied to every segment, and overwrites any existing Languages required score.
Auto-score "Local presence required" from OECD FDI Restrictiveness ():
How this works FDIRRI runs 0 (fully open to foreign investment) to 1 (fully closed) — this is the economy-wide score, not asset-management-specific (the sector-level fund-management score turned out to be nearly uniform across countries, so it can't discriminate; see fdi-restrictiveness.js). Direction is reversed from every other tool here since low restrictiveness is the attractive case: index at or below threshold = 3, above threshold but at or below optimal = 2, above optimal = 1. Hong Kong, UAE, Qatar and Oman aren't covered by this index and are skipped, not guessed at. Same country-level value applied to every segment, and overwrites any existing Local presence required score.
Auto-score "Client servicing" from Mercer CFA Institute Global Pension Index — Integrity ():
How this works Integrity sub-index runs 0-100, higher = stronger pension governance/regulation/communication — direction matches the allocation scorer above (not inverted): below threshold = 1, at or above threshold = 2, at or above optimal = 3. Qatar isn't covered by the Index and is skipped, not guessed at. This same Integrity number was also proposed for "Investor decision-making", but it's only wired up here to avoid two scorecard dimensions moving in lockstep off one source — see mercer-integrity.js. Same country-level value applied to every segment, and overwrites any existing Client servicing score.
Auto-score "Consultant reliant" from IC Research consultant headcount ():
How this works Not a published index like the other sources here — a one-off headcount export from Peter's IC Research Global Advisory Board access (see consultant-density.js), normalized against each country's total tracked AUM to get consultants per $1tn AUM, since raw headcount is dominated by where the big global consulting firms are headquartered (the US and UK alone account for over 15,000 of the ~19,800 individuals in the export). Direction is reversed like the Local presence required tool above, since a denser consultant channel means more distribution resource is needed to reach asset owners: at or below threshold = 3 (low reliance), above threshold but at or below optimal = 2, above optimal = 1 (high reliance). Argentina, Chile, Iceland, Oman, Peru and Qatar aren't in the export and are skipped, not guessed at. Same country-level value applied to every segment, and overwrites any existing Consultant reliant score.
Auto-score "Pricing impact" from segment AUM size:
How this works A starting-point heuristic, not a real pricing dataset — no external source for this exists, so it uses each segment's own total AUM as a rough proxy for fee-negotiating leverage. Direction is reversed like the allocation scorer's opposite: bigger segments can push harder on pricing, so more AUM means more pricing pressure and a worse (lower) score: at or below threshold = 3 (least pressure), above threshold but at or below optimal = 2, above optimal = 1 (most pressure). Segments with no aum_bn are skipped. Overwrites any existing Pricing impact score.
Auto-score "Outsourced management" from default scores by segment type:
How this works No external index exists for this — these are starting-point judgment calls by segment type (same value for e.g. every "DC Pension (Corp)" segment worldwide), not anything derived from Atlas's own data. See segment-type-defaults.js for the reasoning behind each default. Edit any box above before applying if a value looks wrong for your view of the market. Every segment matching a listed type gets that score, in every loaded country; segment types not listed here are left untouched. Overwrites any existing Outsourced management score on matched segments.
